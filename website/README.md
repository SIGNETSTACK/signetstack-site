# SignetStack Labs — corporate website (the built site)

This folder is the **generated, published** site for **Signet Stack Ltd**, built
around the **SignetStack Labs™** master brand. It houses the **Signet Data Trust
Network Platform™** and its modules, the product brands (**SignetStack Velocity™**,
**SignetStack DXP™**), the flagship **HFT V5 Omni™**, and **Signetify™** as a
sister company. Pure HTML/CSS — host on any static host (GitHub Pages, Netlify,
S3/CloudFront, …).

> Generated by `../assets/build_site.py`. **Do not hand-edit the HTML here** — it
> is overwritten on every build. Edit content in `assets/build_site.py`. See the
> root [`README.md`](../README.md) and [`CONTRIBUTING.md`](../CONTRIBUTING.md).

## Run locally
```bash
python3 -m http.server 8099 --directory website
# open http://localhost:8099
```

25 pages total.

## Design system
- Dark **Carbon** theme (`#0B0F14`), **IBM Plex Sans/Mono** (Google Fonts).
- Master/chrome is neutral (white/platinum); each page sets its own `--accent`
  (Velocity cyan, DXP coral, the platform deep-teal, Signetify emerald, …).
- The nested-hex **signet seal** is the master mark; per-domain marks carry a
  distinct glyph.

## Adding a brand or module
See the root [`README.md`](../README.md#adding-a-brand-or-module-the-scalable-part)
— add a mark, add a `DIV`/`MODULES` entry, rerun the build; nav, hubs, footer and
the new themed page are generated automatically.

## Notes / before launch
- **Signetify** links out to [signetify.com](https://signetify.com) (its own brand).
- Contact form is a front-end demo (no backend) — wire to a form handler/email.
- `privacy.html` / `terms.html` are templates — have counsel review.
- Emails: `info@signetstack.io` (general), `johnson@signetstack.io`
  (founder/partnerships/careers). Registered office (UK trading-disclosure
  requirement) is shown in the footer. Trademark clearance on the SignetStack
  names is still advisable.
- Fonts load from the Google Fonts CDN (needs internet); to self-host, drop
  IBM Plex files in `assets/fonts/` and swap the `<link>` for `@font-face`.
- Deployment, DNS and mail records: see [`DEPLOY.md`](DEPLOY.md).
